# PatentAgent v1.1.0 Tauri Packaging Follow-Up

This checklist is manual. Do not publish a tag, GitHub release, DMG, signing result, notarization result, or auto-update channel until each item has live evidence from the current `release/v1.1.0` head.

## Scope

- DMG generation for the Tauri v2 desktop shell.
- macOS signing.
- Apple notarization.
- Optional auto-update design.
- Evidence capture for release notes.

## Manual Gates

1. Confirm `scripts/v1_smoke.sh` passes with the Tauri checks enabled.
2. Confirm the Python environment includes the `.[packaging]` extra so `python3 -m PyInstaller` is available.
3. Run the Tauri build command from the repository root.
4. Record the DMG path, size, and SHA256.
5. Run `codesign --verify --deep --strict` against the generated app bundle.
6. Run `hdiutil verify` against the DMG.
7. Apply [`dmg-ui-regression-guard.md`](./dmg-ui-regression-guard.md) before handing the DMG to a user for visual review.
8. Run notarization only with an explicit release credential decision.
9. Add auto-update only after a signed/notarized artifact exists and the update manifest policy is reviewed.

## Not Completed In Planning

- Do not publish a GitHub release from this checklist alone.
- Do not claim DMG completion before an artifact exists.
- Do not claim the UI is current from specs, screenshots, or OpenDesign artifacts alone; verify the packaged React app.
- Do not claim signing completion before `codesign` verification passes.
- Do not claim notarization completion before Apple notary evidence exists.
- Do not enable auto-update without a separate reviewed implementation.
