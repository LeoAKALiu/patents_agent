## CI bootstrap finding

The first GitHub Actions backend job failed because modern setuptools rejected editable install auto-discovery in the flat repo layout (`backend/` and `frontend/` both top-level). This PR adds explicit package discovery in `pyproject.toml`:

```toml
[tool.setuptools.packages.find]
include = ["backend*"]
```

This keeps the Python distribution scoped to backend packages and allows `python -m pip install -e ".[dev]"` to work in CI.
