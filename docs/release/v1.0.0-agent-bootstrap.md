# PatentAgent v1.0.0 Agent Pipeline Bootstrap

This document describes the conservative bootstrap for the `v1.0.0` agent pipeline. It pairs with:

- `scripts/bootstrap_v1_agent_pipeline.py`
- GitHub issue #14
- release branch `release/v1.0.0`
- Kanban board `patents-v1`

The helper is intentionally idempotent and safe to re-run. It prepares labels, Hermes profile shells, and the Kanban board, but it does not configure secrets, enable auto-merge, or dispatch workers.

## Safety boundaries

The helper must not:

- read or write `.env`, token files, auth stores, or provider keys;
- run `gh auth login`, `hermes auth`, `hermes model`, or `hermes config set ...api_key...`;
- enable or request GitHub auto-merge;
- run `hermes kanban dispatch`;
- create implementation tasks that could be picked up by the dispatcher before human review.

The helper may:

- create/update GitHub labels with `gh label create --force`;
- create missing Hermes profiles with descriptions only;
- run `hermes kanban init`;
- create or switch to the `patents-v1` board;
- print safe next steps for a human maintainer.

## Usage

Dry-run everything first:

```bash
python3 scripts/bootstrap_v1_agent_pipeline.py --no-detect
```

Dry-run against the live local environment and current repo:

```bash
python3 scripts/bootstrap_v1_agent_pipeline.py \
  --repo LeoAKALiu/patents_agent \
  --default-workdir /Users/leo/Projects/patents_agent
```

Apply after reviewing the plan:

```bash
python3 scripts/bootstrap_v1_agent_pipeline.py \
  --apply \
  --repo LeoAKALiu/patents_agent \
  --default-workdir /Users/leo/Projects/patents_agent
```

Apply one component at a time:

```bash
python3 scripts/bootstrap_v1_agent_pipeline.py --apply --component labels --repo LeoAKALiu/patents_agent
python3 scripts/bootstrap_v1_agent_pipeline.py --apply --component profiles
python3 scripts/bootstrap_v1_agent_pipeline.py --apply --component board --default-workdir /Users/leo/Projects/patents_agent
```

Notes:

- `gh auth status` must already pass before `--apply --component labels`; the helper will not authenticate for you.
- Existing Hermes profiles are reported as `exists` and are not cloned or overwritten.
- The default board workdir is the current git worktree root. Pass `--default-workdir /Users/leo/Projects/patents_agent` when running from a temporary release worktree.

## Hermes profiles

The helper creates profile shells with descriptions only. It deliberately avoids `--clone` so it does not copy `.env` or provider credentials from another profile.

| Profile | Role |
|---|---|
| `codexplanner` | Release architecture, task decomposition, branch strategy, complex merge/rebase, and merge-readiness review |
| `claudeissues` | Issue writing, acceptance criteria, UX copy, README/CHANGELOG, and release-note drafting |
| `deepseekworker` | FastAPI backend, parsing, export gates, pytest failures, and deterministic implementation work |
| `qwenworker` | React, TypeScript, Vite, Electron UI, and desktop UX implementation |
| `kimiworker` | Long-context Chinese patent text review, sample drafts, domain prompts, and legal-text consistency checks |
| `codexreviewer` | Final PR review, CI interpretation, merge policy, and release risk assessment |

After this PR is reviewed, configure models and credentials manually through the normal Hermes flows if needed:

```bash
hermes --profile <profile> model
hermes --profile <profile> auth
hermes --profile <profile> tools
```

Do not put provider keys in the repo, issue bodies, PR bodies, Kanban comments, or committed config files.

## Kanban board

The release board is:

```text
slug: patents-v1
name: PatentAgent v1.0.0
description: Autonomous agent pipeline for PatentAgent v1.0.0 Electron desktop release
default workdir: /Users/leo/Projects/patents_agent
```

The helper performs the safe equivalent of:

```bash
hermes kanban init
hermes kanban boards create patents-v1 \
  --name "PatentAgent v1.0.0" \
  --description "Autonomous agent pipeline for PatentAgent v1.0.0 Electron desktop release" \
  --default-workdir /Users/leo/Projects/patents_agent \
  --switch
```

If the board already exists, it switches to it instead of recreating it.

