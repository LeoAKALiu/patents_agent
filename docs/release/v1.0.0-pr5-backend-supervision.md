# PatentAgent v1.0.0 — PR5 Backend Supervision Inside Electron

This document pairs with [GitHub issue #19](https://github.com/LeoAKALiu/patents_agent/issues/19) and extends the PR4 Electron shell with local FastAPI backend supervision. The MVP deliberately uses the developer/user's local Python runtime; packaging a standalone backend binary remains deferred.

## Runtime flow

1. Electron main process starts after `app.whenReady()`.
2. `desktop/electron/backend-supervisor.ts` selects a local port:
   - `PATENTAGENT_BACKEND_PORT` if provided;
   - otherwise an available `127.0.0.1` port allocated by Node.
3. The supervisor spawns the backend with the configured Python executable. On macOS it first checks common local Python locations (including Homebrew/Anaconda) before falling back to `python3` on PATH:

   ```text
   <python> -m uvicorn backend.app.main:app --host 127.0.0.1 --port <port> --log-level warning
   ```

   `PATENTAGENT_PYTHON` can override the executable.
4. The main process polls `http://127.0.0.1:<port>/api/health` until it returns HTTP 200 with `ok=true`.
5. Only after backend health succeeds does Electron load the renderer:
   - dev mode: `PATENTAGENT_VITE_URL` or `http://127.0.0.1:5173`;
   - production mode: `frontend/dist/index.html`.
6. Renderer `/api/*` requests are redirected to the supervised backend with `session.defaultSession.webRequest.onBeforeRequest`.
7. App quit sends `SIGTERM` to the backend child process; a forced kill is available through the supervisor's graceful stop helper.

## User-visible failure path

If the Python process fails to spawn or `/api/health` does not become healthy before the startup timeout, the app opens a dark error page instead of a blank renderer. The page includes the attempted command, health URL, and captured backend output plus the usual local dependency repair command:

```bash
python3 -m pip install -e ".[dev]"
```

## Configuration knobs

| Environment variable | Purpose |
| --- | --- |
| `PATENTAGENT_PYTHON` | Absolute or PATH-resolved Python executable to use instead of `python3`/`python`. |
| `PATENTAGENT_BACKEND_PORT` | Fixed backend port. If omitted, Electron allocates an available local port. |
| `PATENTAGENT_BACKEND_TIMEOUT_MS` | Startup health-check timeout; defaults to 20 seconds. |
| `PATENTAGENT_BACKEND_LOG_LEVEL` | uvicorn log level; defaults to `warning`. |
| `PATENTAGENT_BACKEND_DATA_DIR` | Desktop backend data directory; defaults to Electron `userData/backend-data`. |
| `PATENTAGENT_VITE_URL` | Vite dev server URL for Electron dev mode. |

## Smoke test

`cd desktop && npm run smoke` now verifies both pieces of the desktop contract:

- the Electron main process starts the backend and logs `[smoke] backend health ok: ...` after `/api/health` passes;
- a hidden BrowserWindow still loads the preload and exposes `window.desktop`.

The smoke path uses a temporary backend data directory so it does not write application data into the repository.

## Safety boundaries

PR5 does not package the backend binary, implement secret handling, add native file dialogs, change GitHub merge policy, or push to `main`. It also does not edit local environment files.
