# PatentAgent v1.1.0 Tauri Release Gate

This gate is for validating a local Tauri DMG from an exact `origin/release/v1.1.0` SHA. Record that SHA before building; do not treat the moving remote branch name as release evidence by itself. This gate does not publish a release, create a tag, or claim formal macOS distribution readiness.

## Required Checks Before Marking A Local Tauri DMG Validated

Run from a clean checkout based on the exact release-branch SHA under validation:

```bash
git rev-parse HEAD
npm --prefix frontend ci
npm exec --yes --package @tauri-apps/cli@^2 -- tauri build
hdiutil verify src-tauri/target/release/bundle/dmg/PatentAgent_1.1.0_aarch64.dmg
shasum -a 256 src-tauri/target/release/bundle/dmg/PatentAgent_1.1.0_aarch64.dmg
python3 scripts/tauri_dmg_smoke.py src-tauri/target/release/bundle/dmg/PatentAgent_1.1.0_aarch64.dmg --keep-artifacts
SMOKE_APP=/private/tmp/<tauri-dmg-smoke-dir>/PatentAgent.app
PATENTAGENT_TAURI_DOM_SMOKE=1 \
  PATENTAGENT_TAURI_DOM_SMOKE_REPORT=/tmp/patentagent-tauri-dom-smoke.json \
  "$SMOKE_APP/Contents/MacOS/patentagent-tauri"
scripts/v1_smoke.sh
```

## What This Proves

- The DMG image validates with `hdiutil verify`.
- `PatentAgent.app` can be copied out of the DMG.
- Bundle metadata is present and internally sane: `Info.plist`, executable, identifier, version, and display name.
- The copied app contains `_CodeSignature/CodeResources`.
- `codesign --verify --deep --strict --verbose=4` passes for the copied app, proving local bundle signature integrity for this artifact.
- The copied app launches.
- The Python FastAPI sidecar starts as a child process for the app.
- `/api/health` returns `ok: true`.
- A normal app quit stops both the Tauri app and Python backend sidecar.
- DMG detach succeeds and the detach result is recorded in the smoke summary.

## What This Does Not Prove

- Developer ID signing is not complete while the app is ad-hoc signed with Tauri `signingIdentity: "-"`.
- `codesign_strict_ok: true` does not prove Developer ID signing, hardened runtime, notarization, or Gatekeeper readiness.
- Notarization is not complete unless notarization and stapling evidence is recorded and `spctl` accepts the app.
- On some local macOS runners, `spctl` can return a transient assessment-tool error such as `Too many open files`; the smoke script records this separately and can continue launch validation, but this is not Gatekeeper acceptance.
- DOM-level React renderer validation is separate from `scripts/tauri_dmg_smoke.py`; run the env-gated Tauri DOM smoke before claiming packaged renderer health.
- The Python sidecar is not self-contained while the app depends on host `python3` or another host Python executable.
- Auto-update is not configured.
- No tag, GitHub release, artifact upload, auto-merge, or auto-release is created by these checks.

## Required Evidence In The Release Report

- `origin/release/v1.1.0` SHA.
- DMG path.
- DMG size.
- SHA256.
- `hdiutil verify` result.
- `scripts/tauri_dmg_smoke.py` summary path.
- `scripts/tauri_dmg_smoke.py` key fields:
  - `smoke_dir`
  - `dmg`
  - `bundle_metadata_ok`
  - `codesign_strict_ok`
  - `detach.ok`
  - `spctl.status`
  - `spctl.returncode`
  - `health.ok`
  - `health.llm_configured`
  - `app_alive_after_quit`
  - `backend_alive_after_quit`
- `scripts/v1_smoke.sh` result, including the log and whether any `PATENTAGENT_SKIP_*` environment flag or `Skipping ...` path occurred.
- Tauri DOM smoke report path and key renderer assertions: root, `.app-shell`, `.sidebar`, and `.topbar`.
- Explicit signing and notarization status.
