# PatentAgent v1.0.0 — PR4 Electron Desktop Runtime Skeleton

This document pairs with [GitHub issue #18](https://github.com/LeoAKALiu/patents_agent/issues/18) and adds the Electron desktop shell to `release/v1.0.0`. It is intentionally minimal: the goal is a working BrowserWindow, a typed preload, a native menu, and a headless smoke test that downstream PRs (backend supervision, settings, file dialogs) can build on.

## Workspace layout

```text
desktop/
├── package.json         # electron + typescript dev deps
├── tsconfig.json        # CommonJS output to dist-electron/
├── .gitignore           # node_modules, dist-electron, build outputs
├── electron/
│   ├── main.ts          # main process: window, menu, IPC, smoke
│   └── preload.ts       # contextBridge → window.desktop
└── scripts/
    └── smoke.mjs        # headless launch + preload probe
```

## Safety boundaries

PR4 must not:

- read or write `.env`, `auth.json`, or any credential file;
- enable GitHub auto-merge or push to `main`;
- wire up backend supervision, secret handling, or native file dialogs (PR5+);
- modify existing backend or frontend code beyond the new `desktop/` workspace.

## Renderer loading

- **Development**: `process.env.PATENTAGENT_ELECTRON_DEV=1` (set by `npm run dev`) loads `http://127.0.0.1:5173` so the Vite dev server can serve the renderer with HMR.
- **Production**: loads `../../frontend/dist/index.html` relative to the compiled main entry. Run `cd frontend && npm run build` first; the relative path is stable because both `frontend/dist/index.html` and `desktop/dist-electron/main.js` are siblings of the repo root.

The renderer is locked down with `contextIsolation: true`, `nodeIntegration: false`, `sandbox: true`, and `setWindowOpenHandler(() => ({action: "deny"}))`. Any non-allow-listed navigation is cancelled and routed to the system browser via `shell.openExternal`.

## Native menu

The menu is built in `buildAppMenu()` and registered after `app.whenReady()`. PR4 ships the following actions, with the user-facing labels in Chinese to match the existing UI:

| Menu path            | Action key              | Implementation    |
| -------------------- | ----------------------- | ----------------- |
| App → Quit (macOS)   | `role: "quit"`          | Built-in role     |
| File → 导出目录…     | `open-export-folder`    | IPC stub (PR7)    |
| File → 设置…         | `open-settings`         | IPC stub (PR6)    |
| Help → 关于 PatentAgent | about dialog         | `dialog.showMessageBox` |

Actions for which PR4 is not yet responsible emit a `desktop:menu` IPC message that the renderer (or a follow-up PR) can subscribe to via `window.desktop.onMenuAction(handler)`. The preload returns an unsubscribe function for React-style cleanup.

## Smoke test

The smoke test boots a hidden BrowserWindow, loads an inline data URL (so it doesn't need a built frontend), waits for `did-finish-load`, then probes the renderer for the `window.desktop` API exposed by the preload. It exits 0 on success, 1 on failure, and 124 on a 30-second timeout (configurable via `PATENTAGENT_SMOKE_TIMEOUT_MS`).

```bash
cd desktop
npm install            # one-time, installs electron + typescript
npm run smoke          # builds, then launches electron with --smoke
```

Expected output:

```text
[smoke] preload exposed desktop API: platform=darwin
smoke: passed
```

On Linux CI without a display, wrap with `xvfb-run -a` if you need a real launch; otherwise the `tsc -p tsconfig.json --noEmit` step in `tests/test_desktop_electron_skeleton.py` is enough to gate the change.
