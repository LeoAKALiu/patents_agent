# V1 Smoke Sample: Construction Scene Edge-AI Safety Monitoring

## Patent path

从技术想法撰写发明专利（invention）。

## One-line idea

一种施工场景边缘 AI 安全监控方法，在塔吊、临边洞口和人员密集区域的视频流中融合目标检测、跨摄像头轨迹关联和风险规则，输出可追溯的安全告警事件。

## Technical problem

现有施工现场视频监控通常依赖人工轮巡或单点告警，难以在遮挡、多摄像头视角切换和作业状态变化时持续识别人员未戴安全帽、人员进入危险半径、塔吊吊物下方人员聚集等复合风险。系统还需要保留证据帧、风险阈值和处置记录，便于项目安全员复核。

## Technical solution

1. 在边缘设备上接入多路摄像头视频流，按施工区域生成摄像头、危险源和作业班组的映射表。
2. 对每路视频执行轻量化目标检测，识别人员、安全帽、反光背心、吊钩、吊物和临边防护构件。
3. 结合跨视域 re-id 特征和时间窗口，对同一人员在相邻摄像头中的轨迹进行关联，形成连续作业轨迹。
4. 根据危险源空间半径、轨迹停留时间、目标置信度和工况标签计算风险评分，超过阈值时生成告警事件。
5. 将告警事件与证据帧、检测框、轨迹片段和处置状态一起写入本地数据包，支持离线复核和后续上传。

## Evidence state

- 已具备：YOLO/Ultralytics 检测基线、Jetson Orin Nano 部署经验、施工安全视频监控项目场景。
- 可行未验证：跨摄像头 re-id 在夜间和强遮挡场景下的稳定性。
- 需实验：不同危险源半径和风险阈值对误报率、漏报率的影响。

## Release-smoke expectations

该样例用于 v1 接受性测试中的发明专利路径：创建项目、必要时生成核心公式包、注入严格会审记录、生成初稿、运行质量检查、编译正式稿并通过成稿会审导出。测试不得调用真实 LLM API，也不得依赖任何 `.env` 或凭证。
