# **🔍 核心背景与问题提出**

**自主科研AI的现状**
- **能力边界**：当前自主科研AI Agent已实现"从灵感到论文"的全流程自动化，可独立完成实验执行与论文撰写。
- **关键痛点**：
  1. **系统性错误隐蔽**：生成与审查由同一模型家族完成，难以发现内部一致性错误
  2. **证据支撑模糊**：无人监督下连续工作数天后，结论的证据充分性难以验证
  3. **"AI说谎"风险**：模型可能在结论中植入无证据支持的主张

**ARIS的诞生**
- **全称**：Auto-Research-in-sleep（睡眠中自动研究）
- **定位**：开源科研harness系统（非独立模型）
- **核心目标**：**提升AI生成论文的可检验性**，而非单纯加速写作流程
- **开发团队**：上海交通大学（主要研究者：Ruofeng Yang, Yongcan Li, Shuai Li）

# **🏗️ 系统架构与核心机制**

## **一、三层系统拓扑**

| 层级 | 核心功能 | 关键组件 |
| :--- | :--- | :--- |
| **执行层** | 提供基础能力支撑 | 可复用Markdown技能定义、持久化研究Wiki |
| **编排层** | 串联完整科研流程 | 5个端到端工作流（覆盖发现/实验/成稿/投稿后阶段） |
| **保障层**（核心创新） | 确保研究可信度 | 3阶段证据-声明审计级联、5遍科学写作编辑流水线、数学证明检查器、视觉PDF审查、引用审计 |

## **二、跨模型对抗协作机制**

**"执行-审查-修正"闭环**
- **执行者**：默认Claude家族（Claude Opus 4.6），负责代码生成、实验执行、论文草稿撰写
- **审查者**：默认GPT-5.4家族，按预定义标准评分并返回结构化修改建议
- **协作流程**：生成→评分→修正→再提交，直至达到分数阈值
- **冲突解决**：实验失败时触发Auto-debug（3次重试），仍失败则调用第三模型（/codex:rescue）

**审查者权限等级**
- Medium级：仅文档级访问
- Hard级： artifact感知+记忆功能
- Nightmare级：完整代码仓库访问权限

## **三、五大端到端工作流**

| 工作流ID | 名称 | 核心功能 | 覆盖阶段 |
| :--- | :--- | :--- | :--- |
| W1 | 创意发现 | 文献调研、新颖性检查、实验规划 | 研究发现阶段 |
| W1.5 | 实验桥接 | 代码实现、算力执行、结果回收 | 实验阶段 |
| W2 | 自动审查循环 | 跨模型评分、行动项提取、GPU实验触发、章节修订 | 成稿阶段 |
| W3 | 论文写作 | 7步流程：规划→图表→LaTeX撰写→五轮编辑→证明检查→结论审计→视觉审查 | 成稿阶段 |
| W4 | 投稿后阶段 | 审稿意见解析、回应策略规划、 rebuttal撰写、三道安全检查 | 投稿后阶段 |

# **🔐 证据可信度保障体系**

## **1. 三阶段审计级联**
- **阶段1（实验审计）**：检查实验可靠性，排查伪标签、幽灵结果、未执行指标、超范围外推
- **阶段2（结果-主张映射）**：逐条比对结论与证据，判定"有支持"/"部分支持"/"不成立"
- **阶段3（论文主张审计）**：独立审查者核对文中定量主张与原始结果文件的一致性

## **2. 多层级安全网**
- **科学编辑流水线**：5轮处理（冗余表达→主动语态→局部连贯→术语一致→数字一致）
- **理论审查**：调用数学证明检查器验证理论成分
- **视觉审查**：检查图注错位、版面异常、表格可读性
- **引文核验**：验证文献存在性、元数据正确性及引用与论断的相关性

## **3. 研究知识库（Wiki）**
- **功能**：项目级持久化记录系统，保存论文、想法、实验过程及关联性
- **价值**：
  - 避免重复尝试失败方向（形成"禁试清单"）
  - 已验证结论作为后续研究基础
  - 实现从"反复试错"到"螺旋式学习"的转变

# **⚠️ 局限性与未来方向**

## **当前不足**
- **缺乏受控评估**：所有结果均为观察性记录，无法建立因果关系
- **无法保证绝对正确**：审计链可拦截常见失真，但不能发现所有错误或伪造
- **潜在方法论偏好**：系统可能优化迎合审查者口味而非科学质量
- **安全隐患**：仓库级审查涉及源代码发送外部模型接口，本地审查路由仍在规划中

## **未来发展方向**
- **强化因果验证**：设计对照实验验证跨家族审查的实际优势
- **优化审查机制**：探索多审查者结构及最优协作模式
- **扩展应用场景**：将独立审查、证据审计机制应用于模型自我改进系统
- **增强本地部署**：完善本地审查路由，提升数据安全

# **💡 关键洞察**
- **范式转变**：ARIS将科研自动化的焦点从"速度提升"转向"可信度保障"，填补了行业关键空白
- **人机协作定位**：明确最终研究方向选择、证据核验和提交决定仍需人工负责，构建"AI辅助+人类决策"的合理分工模式
- **开源价值**：作为开源系统，其跨模型协作框架和审计机制可为整个科研AI领域提供可复用的信任基础设施
- **局限性坦诚**：研究团队对结果的克制表述（仅为观察性证据）体现了科学严谨性，避免过度宣传
